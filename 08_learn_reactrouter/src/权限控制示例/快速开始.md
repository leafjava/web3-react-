## 🎯 核心代码片段

### 1. 路由守卫（最基础）

```javascript
function RequireAuth({ children }) {
  const token = getToken()
  
  if (!token) {
    return <Navigate to="/login" replace />
  }
  
  return children
}

// 使用
<Route path="/admin" element={
  <RequireAuth>
    <AdminPage />
  </RequireAuth>
} />
```

---

### 2. 权限组件（最常用）

```javascript
function PermissionButton({ permission, children, ...props }) {
  const { hasPermission } = useAuth()
  
  if (!hasPermission(permission)) {
    return null
  }
  
  return <button {...props}>{children}</button>
}

// 使用
<PermissionButton permission="user:add">
  添加用户
</PermissionButton>
```

---

### 3. 自定义 Hook（最灵活）

```javascript
function useHasPermission(permission) {
  const { permissions } = useAuth()
  return permissions.includes(permission)
}

// 使用
const canAdd = useHasPermission('user:add')
{canAdd && <button>添加用户</button>}
```

---

### 4. Context 管理（最核心）

```javascript
const AuthContext = createContext()

function AuthProvider({ children }) {
  const [user, setUser] = useState(null)
  
  const hasPermission = (permission) => {
    return user?.permissions.includes(permission)
  }
  
  return (
    <AuthContext.Provider value={{ user, hasPermission }}>
      {children}
    </AuthContext.Provider>
  )
}

function useAuth() {
  return useContext(AuthContext)
}
```

## 💡 常见错误

### 错误 1：忘记包裹 AuthProvider

```javascript
// ❌ 错误
<App />

// ✅ 正确
<AuthProvider>
  <App />
</AuthProvider>
```

---

### 错误 2：只在前端控制权限

```javascript
// ❌ 错误：只在前端隐藏按钮
<PermissionButton permission="user:delete">删除</PermissionButton>

// ✅ 正确：前端 + 后端双重验证
<PermissionButton permission="user:delete" onClick={deleteUser}>
  删除
</PermissionButton>

async function deleteUser(id) {
  // 后端也会验证权限
  const response = await fetch(`/api/user/${id}`, {
    method: 'DELETE',
    headers: { Authorization: `Bearer ${token}` }
  })
}
```

---

### 错误 3：权限码硬编码

```javascript
// ❌ 错误：硬编码
<PermissionButton permission="user:add">添加</PermissionButton>

// ✅ 正确：统一管理
export const PERMISSIONS = {
  USER: {
    ADD: 'user:add',
    EDIT: 'user:edit',
    DELETE: 'user:delete'
  }
}

<PermissionButton permission={PERMISSIONS.USER.ADD}>添加</PermissionButton>
```

## 🔧 调试技巧

### 1. 查看当前用户权限

```javascript
function DebugPanel() {
  const { user } = useAuth()
  
  if (process.env.NODE_ENV !== 'development') {
    return null
  }
  
  return (
    <div style={{ position: 'fixed', bottom: 0, right: 0, background: 'white', padding: 10 }}>
      <h4>调试信息</h4>
      <p>用户：{user?.username}</p>
      <p>角色：{user?.role}</p>
      <p>权限：{user?.permissions.join(', ')}</p>
    </div>
  )
}
```

---

### 2. 权限检查日志

```javascript
function hasPermission(permission) {
  const result = user.permissions.includes(permission)
  
  if (process.env.NODE_ENV === 'development') {
    console.log(`[Permission Check] ${permission}: ${result}`)
    if (!result) {
      console.log('User permissions:', user.permissions)
    }
  }
  
  return result
}
```

## 🆘 遇到问题？

### 常见问题

**Q: 路由守卫不生效？**
A: 检查是否正确包裹了 AuthProvider

**Q: 权限组件不显示？**
A: 检查用户是否有对应权限，查看控制台日志

**Q: 登录后跳转失败？**
A: 检查 Navigate 组件的 state 和 location

**Q: 菜单不更新？**
A: 检查是否在权限变更后重新过滤了路由
