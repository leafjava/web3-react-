**面试官您好，这个问题我在实际项目中实现过。首先说一下 Vue 和 React 的对比：**

> **Vue 的实现方式：**
> 
> 1. **路由守卫（页面级）**
> ```javascript
> router.beforeEach((to, from, next) => {
>   if (to.meta.requiresAuth && !token) {
>     next('/login')
>   } else {
>     next()
>   }
> })
> ```

> 2. **自定义指令（按钮级）**
> ```vue

> ```
> 
> **React 的实现方式：**
> 
> 1. **路由守卫（页面级）**
> ```javascript

> <Route path="/admin" element={
>   <RequireAuth>
>     <AdminPage />
>   </RequireAuth>
> } />
> ```
> 
> 2. **按钮级权限**
> ```javascript
> // 使用权限组件或 Hook
> <PermissionButton permission="user:add">添加用户</PermissionButton>
> // 或
> const canAdd = useHasPermission('user:add')
> {canAdd && <button>添加用户</button>}
> ```

### 第二步：说明实现方案（分层次）

**我会根据项目需求实现三个层级的权限控制：**

#### 🔹 第一层：页面级权限（路由守卫）

**实现方式：高阶组件 + React Router v6 的 Navigate**

```javascript
// 1. 登录验证
function RequireAuth({ children }) {
  const token = getToken()
  const location = useLocation()
  
  if (!token) {
    return <Navigate to="/login" state={{ from: location }} replace />
  }
  
  return children
}

// 2. 角色验证
function RequireRole({ children, allowedRoles }) {
  const { role } = useAuth()
  
  if (!allowedRoles.includes(role)) {
    return <Navigate to="/403" replace />
  }
  
  return children
}

// 3. 权限码验证
function RequirePermission({ children, permission }) {
  const { hasPermission } = useAuth()
  
  if (!hasPermission(permission)) {
    return <Navigate to="/403" replace />
  }
  
  return children
}

// 使用
<Route path="/admin" element={
  <RequireRole allowedRoles={['admin']}>
    <AdminPage />
  </RequireRole>
} />
```

**优点：**
- 类似 Vue 的路由守卫，容易理解
- 保护路由，未授权用户无法访问
- 可以保存跳转前的路径，登录后返回

---

#### 🔹 第二层：模块级权限（动态路由和菜单）

**实现方式：根据用户权限过滤路由配置**

```javascript
// 1. 定义路由配置（包含权限信息）
const routeConfig = [
  {
    path: '/user',
    element: <UserPage />,
    meta: {
      title: '用户管理',
      icon: '👥',
      permissions: ['user:list']
    },
    children: [
      {
        path: 'list',
        element: <UserList />,
        meta: { title: '用户列表', permissions: ['user:list'] }
      },
      {
        path: 'add',
        element: <UserAdd />,
        meta: { title: '添加用户', permissions: ['user:add'] }
      }
    ]
  }
]

// 2. 根据权限过滤路由
function filterRoutes(routes, userPermissions) {
  return routes.filter(route => {
    // 检查权限
    if (route.meta?.permissions) {
      const hasPermission = route.meta.permissions.some(p => 
        userPermissions.includes(p)
      )
      if (!hasPermission) return false
    }
    
    // 递归过滤子路由
    if (route.children) {
      route.children = filterRoutes(route.children, userPermissions)
    }
    
    return true
  })
}

// 3. 生成动态菜单
function generateMenu(routes) {
  return routes.map(route => ({
    path: route.path,
    title: route.meta?.title,
    icon: route.meta?.icon,
    children: route.children ? generateMenu(route.children) : []
  }))
}
```

**优点：**
- 菜单和路由自动同步
- 无权限的路由不会显示
- 配置化管理，易于维护

#### 🔹 第三层：按钮级权限（细粒度控制）

**实现方式：权限组件 + 自定义 Hook + Context**

```javascript
// 1. 权限 Context
const AuthContext = createContext()

function AuthProvider({ children, userInfo }) {
  const hasPermission = (permission) => {
    return userInfo.permissions.includes(permission)
  }
  
  return (
    <AuthContext.Provider value={{ userInfo, hasPermission }}>
      {children}
    </AuthContext.Provider>
  )
}

// 2. 权限组件（推荐）
function PermissionButton({ permission, children, ...props }) {
  const { hasPermission } = useContext(AuthContext)
  
  if (!hasPermission(permission)) {
    return null // 无权限不显示
  }
  
  return <button {...props}>{children}</button>
}

// 3. 自定义 Hook（灵活）
function useHasPermission(permission) {
  const { hasPermission } = useContext(AuthContext)
  return hasPermission(permission)
}

// 使用方式 1：权限组件
<PermissionButton permission="user:add">添加用户</PermissionButton>

// 使用方式 2：自定义 Hook
const canAdd = useHasPermission('user:add')
{canAdd && <button>添加用户</button>}

// 使用方式 3：通用权限组件
<Permission permission="user:delete" fallback={<span>无权限</span>}>
  <button>删除用户</button>
</Permission>
```

**优点：**
- 类似 Vue 的 v-permission 指令
- 灵活，支持多种使用方式
- 可以提供 fallback 提示

### 第四步：说明权限粒度和命名规范

**权限粒度设计：**

| 级别 | 说明 | 示例 | 使用场景 |
|-----|------|------|---------|
| 角色级 | 粗粒度 | admin, user, guest | 区分用户类型 |
| 模块级 | 中粒度 | user:*, order:* | 控制模块访问 |
| 操作级 | 细粒度 | user:add, user:delete | 控制具体操作 |

**权限码命名规范：**

```
格式：模块:操作

示例：
- user:list    （查看用户列表）
- user:add     （添加用户）
- user:edit    （编辑用户）
- user:delete  （删除用户）
- order:list   （查看订单）
- order:export （导出订单）
```

### 1. 安全性考虑

**前端权限只是 UI 控制，后端必须验证！**

```javascript
// ❌ 错误：只在前端控制
<button onClick={deleteUser}>删除</button>

// ✅ 正确：前端 + 后端双重验证
<PermissionButton permission="user:delete" onClick={deleteUser}>
  删除
</PermissionButton>

// 后端也要验证
async function deleteUser(id) {
  const response = await fetch(`/api/user/${id}`, {
    method: 'DELETE',
    headers: { Authorization: `Bearer ${token}` }
  })
  // 后端会验证 token 和权限
}
```

### 4. 权限刷新

// 定期刷新权限（防止权限变更后用户仍能操作）
useEffect(() => {
  const refreshPermissions = async () => {
    const response = await fetch('/api/user/permissions')
    const { permissions } = await response.json()
    updatePermissions(permissions)

### 5. 权限缓存

```javascript
// 使用 localStorage 缓存权限，减少请求
const CACHE_KEY = 'user_permissions'
const CACHE_TIME = 5 * 60 * 1000 // 5 分钟

function getPermissionsFromCache() {
  const cache = localStorage.getItem(CACHE_KEY)
  if (!cache) return null
  
  const { permissions, timestamp } = JSON.parse(cache)
  
  // 检查是否过期
  if (Date.now() - timestamp > CACHE_TIME) {
    return null
  }
  
  return permissions
}

function setPermissionsToCache(permissions) {
  localStorage.setItem(CACHE_KEY, JSON.stringify({
    permissions,
    timestamp: Date.now()
  }))
}
```
