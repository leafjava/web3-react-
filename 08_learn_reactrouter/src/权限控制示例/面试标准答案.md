# 面试标准答案：React 权限控制

## 问题：如何实现按钮级 / 页面级权限控制？

---

## 标准回答（3-5 分钟）

### 开场（30秒）

面试官您好，这个问题我在实际项目中实现过。React 的权限控制和 Vue 的思路类似，但实现方式不同。Vue 使用路由守卫和自定义指令，React 则使用高阶组件和权限组件。我会从三个层级来说明。

---

### 第一部分：页面级权限（1分钟）

**实现方式：路由守卫 + Navigate 组件**

在 Vue 中我们用 `router.beforeEach`，在 React 中我用高阶组件包裹路由：

```javascript
// 创建路由守卫组件
function RequireAuth({ children }) {
  const token = localStorage.getItem('token')
  const location = useLocation()
  
  if (!token) {
    // 未登录，跳转到登录页
    return <Navigate to="/login" state={{ from: location }} replace />
  }
  
  return children
}

// 使用
<Route path="/admin" element={
  <RequireAuth>
    <AdminPage />
  </RequireAuth>
} />
```

这样就实现了页面级的权限控制，未登录用户无法访问受保护的页面。

---

### 第二部分：按钮级权限（1.5分钟）

**实现方式：Context + 权限组件**

类似 Vue 的 `v-permission` 指令，我在 React 中创建权限组件：

```javascript
// 1. 创建权限 Context
const AuthContext = createContext()

function AuthProvider({ children, userInfo }) {
  const hasPermission = (permission) => {
    return userInfo.permissions.includes(permission)
  }
  
  return (
    <AuthContext.Provider value={{ userInfo, hasPermission }}>
      {children}
    </AuthContext.Provider>
  )
}

// 2. 创建权限组件
function PermissionButton({ permission, children, ...props }) {
  const { hasPermission } = useContext(AuthContext)
  
  if (!hasPermission(permission)) {
    return null // 无权限不显示
  }
  
  return <button {...props}>{children}</button>
}

// 3. 使用
<PermissionButton permission="user:add">添加用户</PermissionButton>
<PermissionButton permission="user:delete">删除用户</PermissionButton>
```

也可以用自定义 Hook 实现更灵活的控制：

```javascript
function useHasPermission(permission) {
  const { hasPermission } = useContext(AuthContext)
  return hasPermission(permission)
}

// 使用
const canAdd = useHasPermission('user:add')
{canAdd && <button>添加用户</button>}
```

### 第三部分：动态路由（1分钟）

**实现方式：根据权限过滤路由配置**

```javascript
// 1. 定义路由配置（包含权限信息）
const routeConfig = [
  {
    path: '/user',
    element: <UserPage />,
    meta: {
      title: '用户管理',
      permissions: ['user:list']
    }
  },
  {
    path: '/admin',
    element: <AdminPage />,
    meta: {
      title: '系统管理',
      roles: ['admin']
    }
  }
]

// 2. 根据用户权限过滤路由
function filterRoutes(routes, userPermissions) {
  return routes.filter(route => {
    if (route.meta?.permissions) {
      return route.meta.permissions.some(p => 
        userPermissions.includes(p)
      )
    }
    return true
  })
}

// 3. 使用过滤后的路由
const filteredRoutes = filterRoutes(routeConfig, user.permissions)
const element = useRoutes(filteredRoutes)
```

这样菜单和路由会根据用户权限自动显示和隐藏。

### 第四部分：权限粒度设计（30秒）

我会设计三种权限粒度：

1. **角色级**：`admin`、`user`、`guest` - 区分用户类型
2. **模块级**：`user:*`、`order:*` - 控制模块访问
3. **操作级**：`user:add`、`user:delete` - 控制具体操作

权限码命名规范：`模块:操作`，比如 `user:add`、`order:export`

---

### 第五部分：安全性说明（30秒）

**重要提醒：前端权限只是 UI 控制，后端必须验证！**

```javascript
// 前端控制按钮显示
<PermissionButton permission="user:delete" onClick={deleteUser}>
  删除
</PermissionButton>

// 后端也要验证权限
async function deleteUser(id) {
  const response = await fetch(`/api/user/${id}`, {
    method: 'DELETE',
    headers: { Authorization: `Bearer ${token}` }
  })
  // 后端会验证 token 和权限
}
```

前端权限只是提升用户体验，真正的安全保障在后端。

---

### 总结（20秒）

总结一下，React 权限控制分三个层级：

1. **页面级**：用路由守卫（RequireAuth 组件）
2. **按钮级**：用权限组件（PermissionButton）或 Hook
3. **模块级**：用动态路由（过滤路由配置）

核心是用 Context 管理权限状态，用组件和 Hook 控制显示，前后端双重验证保证安全。

## 高频追问及答案

### Q1: 如何处理权限变更？

**答：** 三种方案：

1. **定期刷新**（推荐）：每 5 分钟刷新一次权限
2. **WebSocket 推送**：权限变更时实时推送
3. **操作前验证**：每次操作前向后端验证

```javascript
// 定期刷新
useEffect(() => {
  const timer = setInterval(async () => {
    const { permissions } = await fetch('/api/user/permissions')
    updatePermissions(permissions)
  }, 5 * 60 * 1000)
  
  return () => clearInterval(timer)
}, [])
```

---

### Q2: 如何优化性能？

**答：** 三个方面：

1. **使用 React.memo** 避免不必要的重渲染
2. **使用 useMemo** 缓存权限判断结果
3. **路由懒加载** 按需加载页面组件

```javascript
// 缓存权限判断
const canAdd = useMemo(() => 
  hasPermission('user:add'), 
  [permissions]
)

// 路由懒加载
const UserPage = lazy(() => import('./pages/User'))
```

### Q3: 如何测试权限控制？

**答：** 三种测试：

1. **单元测试**：测试权限判断逻辑
2. **集成测试**：测试路由守卫
3. **E2E 测试**：测试完整流程

```javascript
// 单元测试示例

  const { result } = renderHook(() => useHasPermission('user:add'))
  expect(result.current).toBe(true)
})
```
