# React 多步骤表单方案对比

## 📊 基础方案 vs 进阶方案

| 对比维度 | 基础方案 | 进阶方案 |
|---------|---------|---------|
| **适用场景** | 中小型表单（< 10 字段） | 大型复杂表单（10+ 字段） |
| **学习成本** | ⭐⭐ 低 | ⭐⭐⭐⭐ 中高 |
| **开发速度** | ⚡⚡⚡ 快 | ⚡⚡ 中等 |
| **代码量** | 少（~300 行） | 多（~500 行） |
| **可维护性** | ⭐⭐⭐ 中等 | ⭐⭐⭐⭐⭐ 优秀 |
| **性能** | ⭐⭐⭐ 良好 | ⭐⭐⭐⭐⭐ 优秀 |
| **可复用性** | ⭐⭐ 低 | ⭐⭐⭐⭐⭐ 高 |
| **类型安全** | ❌ 无 | ✅ 可配合 TS |
| **测试难度** | ⭐⭐⭐ 中等 | ⭐⭐ 容易 |

## 🔍 详细对比

### 1. 状态管理

#### 基础方案
```javascript
// 类组件 + this.state
this.state = {
  currentStep: 1,
  tradingPair: 'ETH/USDT',
  direction: 'long',
  leverage: 10,
  collateral: '',
  errors: {}
}

// 统一更新
handleInputChange = (event) => {
  const { name, value } = event.target
  this.setState({ [name]: value })
}
```

**优点：**
- ✅ 简单直观
- ✅ 适合快速原型
- ✅ 符合 CoderWhy 课程教学

**缺点：**
- ❌ 所有状态在一个对象中，难以拆分
- ❌ 类组件写法较繁琐
- ❌ 难以复用逻辑

---

#### 进阶方案
```javascript
// 自定义 Hook 封装
function useFormData(initialData) {
  const [formData, setFormData] = useState(initialData)
  const [errors, setErrors] = useState({})
  
  const updateField = (name, value) => {
    setFormData(prev => ({ ...prev, [name]: value }))
  }
  
  return { formData, errors, updateField }
}

// 使用
const { formData, updateField } = useFormData(initialData)
```

**优点：**
- ✅ 逻辑复用（多个表单共享）
- ✅ 函数组件，代码更简洁
- ✅ 易于测试
- ✅ 易于扩展

**缺点：**
- ❌ 需要理解 Hook 原理
- ❌ 初期开发稍慢

---

### 2. 组件结构

#### 基础方案
```javascript
// 所有步骤在一个组件中
class LeverageTradeForm extends PureComponent {
  renderStep1() { /* ... */ }
  renderStep2() { /* ... */ }
  renderStep3() { /* ... */ }
  renderStep4() { /* ... */ }
  
  render() {
    return (
      <div>
        {currentStep === 1 && this.renderStep1()}
        {currentStep === 2 && this.renderStep2()}
        {/* ... */}
      </div>
    )
  }
}
```

**优点：**
- ✅ 所有逻辑在一个文件
- ✅ 容易理解整体流程
- ✅ 数据共享方便

**缺点：**
- ❌ 单文件过大（500+ 行）
- ❌ 难以并行开发
- ❌ 难以单独测试某个步骤

---

#### 进阶方案
```javascript
// 每个步骤独立组件
function Step1TradingPair({ formData, updateField, errors }) {
  return <div>...</div>
}

function Step2Leverage({ formData, updateField, errors }) {
  return <div>...</div>
}

// 主组件配置化
const steps = [
  { title: '选择交易对', component: Step1TradingPair },
  { title: '设置杠杆', component: Step2Leverage },
]

const CurrentStep = steps[currentStep - 1].component
return <CurrentStep {...props} />
```

**优点：**
- ✅ 职责单一，易于维护
- ✅ 可以并行开发
- ✅ 易于单元测试
- ✅ 可以按需懒加载

**缺点：**
- ❌ 文件数量增多
- ❌ 需要管理组件间通信

---

### 3. 表单验证

#### 基础方案
```javascript
validateCurrentStep = () => {
  const errors = {}
  
  switch (this.state.currentStep) {
    case 1:
      if (!this.state.tradingPair) {
        errors.tradingPair = '请选择交易对'
      }
      break
    case 2:
      if (!this.state.collateral) {
        errors.collateral = '请输入保证金'
      }
      break
  }
  
  this.setState({ errors })
  return Object.keys(errors).length === 0
}
```

**优点：**
- ✅ 简单直接
- ✅ 容易理解

**缺点：**
- ❌ 验证逻辑分散
- ❌ 难以复用
- ❌ 没有类型提示

---

#### 进阶方案（可配合 Zod）
```javascript
import { z } from 'zod'

// 定义验证规则
const step1Schema = z.object({
  tradingPair: z.string().min(1, '请选择交易对'),
  direction: z.enum(['long', 'short'])
})

// 使用
const validate = (data) => {
  try {
    step1Schema.parse(data)
    return { success: true }
  } catch (error) {
    return { success: false, errors: error.errors }
  }
}
```

**优点：**
- ✅ 类型安全
- ✅ 验证规则可复用
- ✅ 错误信息统一
- ✅ 支持复杂验证（异步、跨字段）

**缺点：**
- ❌ 需要学习 Zod API
- ❌ 增加依赖

---

### 4. 性能优化

#### 基础方案
```javascript
// 每次 setState 都会重新渲染整个组件
handleInputChange = (event) => {
  this.setState({ [event.target.name]: event.target.value })
}

// 计算属性每次都重新计算
calculatePositionSize = () => {
  return this.state.collateral * this.state.leverage
}
```

**性能问题：**
- ❌ 输入时整个表单重新渲染
- ❌ 计算属性重复计算
- ❌ 所有步骤组件都在 DOM 中

---

#### 进阶方案
```javascript
// 使用 useMemo 缓存计算结果
const positionSize = useMemo(() => {
  return parseFloat(formData.collateral || 0) * formData.leverage
}, [formData.collateral, formData.leverage])

// 使用 React.memo 避免不必要的重渲染
const Step1 = React.memo(({ formData, updateField }) => {
  return <div>...</div>
})

// 懒加载步骤组件
const Step1 = lazy(() => import('./Step1'))
```

**性能优化：**
- ✅ 只重新渲染变化的部分
- ✅ 计算结果缓存
- ✅ 按需加载组件

## 🎯 选择建议

### 选择基础方案，如果：
- ✅ 表单字段少于 10 个
- ✅ 验证逻辑简单
- ✅ 快速原型开发
- ✅ 团队对 React 不太熟悉
- ✅ 不需要复用到其他地方

### 选择进阶方案，如果：
- ✅ 表单字段超过 10 个
- ✅ 需要复杂验证（异步、跨字段）
- ✅ 需要高性能（大量用户）
- ✅ 需要复用到多个地方
- ✅ 团队使用 TypeScript
- ✅ 长期维护的项目

## 🔄 迁移路径

### 从基础到进阶

**第 1 步：提取自定义 Hook**
```javascript
// 之前
this.state = { formData: {...} }

// 之后
const { formData, updateField } = useFormData(initialData)
```

**第 2 步：拆分步骤组件**
```javascript
// 之前
renderStep1() { return <div>...</div> }

// 之后
function Step1({ formData, updateField }) {
  return <div>...</div>
}
```

**第 3 步：添加性能优化**
```javascript
// 添加 useMemo
const positionSize = useMemo(() => calculate(), [deps])

// 添加 React.memo
export default React.memo(Step1)
```

```javascript
// 安装 Zod
npm install zod

const schema = z.object({...})
```

## 📈 实际项目经验

### 小型项目（< 5 个表单）
**推荐：基础方案**
- 开发快
- 维护简单
- 够用

### 中型项目（5-20 个表单）
**推荐：进阶方案**
- 复用性强
- 统一管理
- 易于维护

### 大型项目（20+ 个表单）
**推荐：进阶方案 + 表单库**
- React Hook Form
- 配置化生成表单
- 统一的验证规则库

## 💡 总结

**没有最好的方案，只有最合适的方案！**

- 🚀 快速开发 → 基础方案
- 🏗️ 长期维护 → 进阶方案

- 先让功能跑起来（基础方案）
