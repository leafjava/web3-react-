### 第一部分：TypeScript 类型安全（1分钟）

**为什么用 TypeScript？**

在 Web3 项目中，合约交互、钱包连接、交易处理都涉及复杂的数据结构，TypeScript 可以在编译时发现错误，避免运行时崩溃。

**核心实践：**

```typescript
// 1. 定义合约类型
interface Token {
  address: `0x${string}`
  symbol: string
  decimals: number
  balance: bigint
}

// 2. 定义 Hook 返回类型
interface UseWalletReturn {
  address: `0x${string}` | undefined
  chainId: number | undefined
  isConnected: boolean
  connect: () => Promise<void>
  disconnect: () => void
}

// 3. 使用泛型增强复用性
function useContractRead<T>(
  address: `0x${string}`,
  abi: any[],
  functionName: string
): {
  data: T | undefined
  isLoading: boolean
  error: Error | null
} {
  // 实现...
}

// 使用
const { data } = useContractRead<bigint>(
  '0x123...',
  ERC20_ABI,
  'balanceOf'
)
```

**优势：**
- 编译时类型检查，减少 bug
- IDE 智能提示，提升开发效率
- 重构更安全，不怕改漏

### 第二部分：自定义 Hook 复用逻辑（1.5分钟）

**类似 Vue 的 Composables，React 用自定义 Hook 提取公共逻辑。**

**核心实践：**

```typescript
// 1. 钱包连接 Hook
function useWallet() {
  const [address, setAddress] = useState<`0x${string}`>()
  const [chainId, setChainId] = useState<number>()
  
  const connect = async () => {
    if (!window.ethereum) throw new Error('请安装 MetaMask')
    
    const accounts = await window.ethereum.request({
      method: 'eth_requestAccounts'
    })
    setAddress(accounts[0])
    
    const chainId = await window.ethereum.request({
      method: 'eth_chainId'
    })
    setChainId(parseInt(chainId, 16))
  }
  
  return { address, chainId, connect, isConnected: !!address }
}

// 2. 合约读取 Hook
function useContractRead<T>(
  address: `0x${string}`,
  abi: any[],
  functionName: string,
  args?: any[]
) {
  const [data, setData] = useState<T>()
  const [isLoading, setIsLoading] = useState(false)
  const [error, setError] = useState<Error | null>(null)
  
  useEffect(() => {
    const fetchData = async () => {
      setIsLoading(true)
      try {
        const contract = new ethers.Contract(address, abi, provider)
        const result = await contract[functionName](...(args || []))
        setData(result)
      } catch (err) {
        setError(err as Error)
      } finally {
        setIsLoading(false)
      }
    }
    
    fetchData()
  }, [address, functionName, JSON.stringify(args)])
  
  return { data, isLoading, error }
}

// 3. 交易处理 Hook
function useTransaction() {
  const [status, setStatus] = useState<'idle' | 'pending' | 'success' | 'error'>('idle')
  const [txHash, setTxHash] = useState<string>()
  
  const sendTransaction = async (tx: TransactionRequest) => {
    setStatus('pending')
    try {
      const signer = await provider.getSigner()
      const response = await signer.sendTransaction(tx)
      setTxHash(response.hash)
      
      await response.wait()
      setStatus('success')
    } catch (error) {
      setStatus('error')
      throw error
    }
  }
  
  return { sendTransaction, status, txHash }
}
```

**使用示例：**

```typescript
function TokenBalance() {
  const { address } = useWallet()
  const { data: balance, isLoading } = useContractRead<bigint>(
    TOKEN_ADDRESS,
    ERC20_ABI,
    'balanceOf',
    [address]
  )
  
  if (isLoading) return <div>加载中...</div>
  return <div>余额: {formatUnits(balance, 18)}</div>
}
```

**优势：**
- 逻辑复用，避免重复代码
- 关注点分离，组件更简洁
- 易于测试和维护

### 第四部分：目录结构规范（1分钟）

**推荐使用 Feature-Sliced Design（FSD）架构**

```
src/
├── app/                    # 应用层（路由、全局状态）
│   ├── providers/          # Context Providers
│   ├── routes/             # 路由配置
│   └── App.tsx
│
├── pages/                  # 页面层
│   ├── home/
│   ├── swap/
│   └── stake/
│
├── widgets/                # 组合层（页面区块）
│   ├── wallet-connect/
│   ├── token-balance/
│   └── transaction-history/
│
├── features/               # 功能层（业务逻辑）
│   ├── swap/
│   │   ├── ui/             # UI 组件
│   │   ├── model/          # 状态管理
│   │   └── api/            # API 调用
│   ├── stake/
│   └── liquidity/
│
├── entities/               # 实体层（业务实体）
│   ├── token/
│   │   ├── model/          # Token 类型定义
│   │   ├── ui/             # Token 相关组件
│   │   └── api/            # Token API
│   ├── user/
│   └── transaction/
│
├── shared/                 # 共享层（通用工具）
│   ├── ui/                 # 通用 UI 组件
│   ├── lib/                # 工具函数
│   ├── hooks/              # 通用 Hooks
│   ├── config/             # 配置文件
│   └── types/              # 通用类型
│
└── contracts/              # 合约层（Web3 特有）
    ├── abis/               # 合约 ABI
    ├── addresses/          # 合约地址
    └── types/              # 合约类型
```

**核心原则：**
- 从上到下依赖（上层可以依赖下层，反之不行）
- 同层不能相互依赖
- 每层职责清晰

### 总结（30秒）

总结一下，保证大型 Web3 项目代码可维护性的五个关键：

1. **TypeScript**：类型安全，编译时发现错误
2. **自定义 Hook**：逻辑复用，类似 Vue 的 Composables
3. **组件库**：UI 统一，推荐 shadcn/ui
4. **目录结构**：FSD 架构，职责清晰
5. **工具链**：ESLint + Prettier + Husky，自动化保证质量

核心思想是：**类型安全 + 逻辑复用 + 结构清晰 + 自动化检查**

## 高频追问及答案

### Q1: 为什么选择 shadcn/ui 而不是 Ant Design？

**答：**

**shadcn/ui 优势：**
- 无样式组件，完全可定制
- 代码直接复制到项目，不是 npm 包
- 基于 Radix UI，可访问性好
- 适合 Web3 项目的现代设计

**Ant Design 优势：**
- 开箱即用，组件丰富
- 中后台项目首选
- 社区成熟，文档完善

**我的选择：**
- Web3 DApp → shadcn/ui（需要定制化）
- 管理后台 → Ant Design（快速开发）

### Q2: FSD 架构和传统 MVC 有什么区别？

**答：**

**传统 MVC：**
```
src/
├── models/      # 所有数据模型
├── views/       # 所有视图
└── controllers/ # 所有控制器
```

**FSD 架构：**
```
src/
├── features/
│   ├── swap/    # Swap 功能的所有代码
│   └── stake/   # Stake 功能的所有代码
└── shared/      # 共享代码
```

**核心区别：**
- MVC：按技术分层（横向切分）
- FSD：按功能分层（纵向切分）

**FSD 优势：**
- 功能内聚，修改一个功能只需改一个目录
- 易于团队协作，不同人负责不同功能
- 易于删除功能，直接删除目录即可

### Q3: 如何测试自定义 Hook？

**答：**

使用 `@testing-library/react-hooks`：

```typescript
import { renderHook, waitFor } from '@testing-library/react'
import { useWallet } from './useWallet'

describe('useWallet', () => {
  test('should connect wallet', async () => {
    const { result } = renderHook(() => useWallet())
    
    expect(result.current.isConnected).toBe(false)
    
    await result.current.connect()
    
    await waitFor(() => {
      expect(result.current.isConnected).toBe(true)
      expect(result.current.address).toBeDefined()
    })
  })
  
  test('should handle connection error', async () => {
    // Mock window.ethereum
    window.ethereum = undefined
    
    const { result } = renderHook(() => useWallet())
    
    await expect(result.current.connect()).rejects.toThrow('请安装 MetaMask')
  })
})
```

### Q4: 如何处理合约 ABI 的类型？

**答：**

使用 `wagmi` 的 `abitype` 或 `typechain`：

```typescript
// 1. 使用 typechain 生成类型
// npx typechain --target ethers-v6 --out-dir src/contracts/types 'src/contracts/abis/*.json'

import { ERC20__factory } from '@/contracts/types'

// 2. 使用生成的类型
const contract = ERC20__factory.connect(address, signer)
const balance = await contract.balanceOf(userAddress) // 类型安全

// 3. 或使用 wagmi 的 abitype
import { parseAbi } from 'abitype'

const abi = parseAbi([
  'function balanceOf(address) view returns (uint256)',
  'function transfer(address to, uint256 amount) returns (bool)'
])

// 类型会自动推导
```
