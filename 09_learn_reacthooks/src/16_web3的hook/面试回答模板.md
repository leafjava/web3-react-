# React Hooks 面试回答模板

## 1. useEffect 的依赖数组原理

### 面试回答模板：

**useEffect 的依赖数组原理主要涉及以下几个方面：**

#### 1.1 基本原理
useEffect 的依赖数组用于控制副作用函数的执行时机。React 会在每次渲染后，通过 **Object.is()** 算法对比依赖数组中的每一项，判断是否发生变化：
- 如果依赖项发生变化，则执行副作用函数
- 如果依赖项未变化，则跳过本次执行

#### 1.2 三种使用场景

```javascript
// 场景1：不传依赖数组 - 每次渲染都执行
useEffect(() => {
  console.log('每次渲染都执行');
});

// 场景2：空依赖数组 - 只在组件挂载时执行一次
useEffect(() => {
  console.log('只在挂载时执行');
}, []);

// 场景3：传入依赖项 - 依赖项变化时执行
useEffect(() => {
  console.log('count 变化时执行');
}, [count]);
```

#### 1.3 对比机制（浅比较）

React 使用 **Object.is()** 进行浅比较，这意味着：
- **基本类型**：比较值是否相同
- **引用类型**：比较引用地址是否相同

```javascript
// ❌ 错误示例：每次都会执行（对象引用每次都不同）
useEffect(() => {
  fetchData(config);
}, [{ id: 1 }]); // 每次渲染都创建新对象

// ✅ 正确示例：使用 useMemo 缓存对象
const config = useMemo(() => ({ id: 1 }), []);
useEffect(() => {
  fetchData(config);
}, [config]);
```

#### 1.4 常见陷阱与最佳实践

**陷阱1：遗漏依赖项**
```javascript
// ❌ 错误：count 应该加入依赖数组
useEffect(() => {
  console.log(count);
}, []); // ESLint 会警告

// ✅ 正确
useEffect(() => {
  console.log(count);
}, [count]);
```

**陷阱2：依赖项是对象或数组**
```javascript
// ❌ 错误：每次渲染都创建新数组
useEffect(() => {
  fetchUsers(userIds);
}, [userIds]); // 如果 userIds 每次都是新数组

// ✅ 正确：使用 JSON.stringify 或 useMemo
useEffect(() => {
  fetchUsers(userIds);
}, [JSON.stringify(userIds)]);
```

#### 1.5 清理函数的执行时机

```javascript
useEffect(() => {
  const timer = setInterval(() => {
    console.log('定时器执行');
  }, 1000);

  // 清理函数在以下时机执行：
  // 1. 组件卸载时
  // 2. 依赖项变化，下次 effect 执行前
  return () => {
    clearInterval(timer);
    console.log('清理定时器');
  };
}, [dependency]);
```

## 2. 自定义 Hook 封装案例

### 面试回答模板：

**我在项目中封装过多个自定义 Hook，主要包括：**

### 2.1 useWallet - 钱包连接管理

**使用场景：** Web3 项目中管理钱包连接状态、账户信息和余额查询

**核心功能：**
- 连接/断开钱包
- 监听账户切换和网络切换
- 查询账户余额
- 错误处理和加载状态管理

**技术要点：**
- 使用 useState 管理连接状态
- 使用 useEffect 监听钱包事件
- 使用 useCallback 优化回调函数
- 自动清理事件监听器

### 2.2 useContractRead - 智能合约读取

**使用场景：** 读取区块链智能合约数据

**核心功能：**
- 自动调用合约只读方法
- 支持轮询刷新
- 缓存管理
- 错误重试机制

**技术要点：**
- 使用 useEffect 处理异步数据获取
- 使用 useMemo 缓存合约实例
- 依赖数组控制重新请求时机
- 防抖/节流优化

### 2.3 useTransaction - 交易管理

**使用场景：** 发送区块链交易并跟踪状态

**核心功能：**
- 发送交易
- 跟踪交易状态（pending/success/failed）
- 等待交易确认
- Gas 费估算

**技术要点：**
- 使用 useReducer 管理复杂状态
- 使用 useCallback 封装交易发送逻辑
- Promise 处理异步流程
- 错误边界处理

## 3. Vue3 Composition API 对比

### 相似之处：

| React Hooks | Vue3 Composition API | 用途 |
|------------|---------------------|------|
| useState | ref / reactive | 响应式状态 |
| useEffect | watchEffect / watch | 副作用处理 |
| useMemo | computed | 计算属性 |
| useCallback | - | 函数缓存 |
| useRef | ref | DOM 引用 |
| 自定义 Hook | 组合式函数 | 逻辑复用 |

### 核心思想一致：
- **逻辑复用**：通过函数组合实现代码复用
- **关注点分离**：按功能组织代码，而非生命周期
- **类型友好**：更好的 TypeScript 支持

## 4. 面试加分项

### 深入理解：
1. **闭包陷阱**：useEffect 中访问的是快照值
2. **性能优化**：合理使用 useMemo 和 useCallback
3. **依赖管理**：使用 ESLint 插件检查依赖完整性
4. **自定义 Hook 命名**：必须以 use 开头
5. **Hook 规则**：只能在顶层调用，不能在条件语句中使用

### 实战经验：
- 在大型项目中封装过 10+ 个自定义 Hook
- 理解 Hook 的底层实现原理（Fiber 链表）
- 能够根据业务场景设计合理的 Hook API
- 注重代码复用性和可维护性
