# Web3 面试题：多链切换和钱包断连处理

## 📝 面试题

**问题：在 Web3 应用中，如何实现多链切换和钱包断连处理？请说明实现原理和注意事项。**

---

## ✅ 标准答案

### 一、核心实现原理

在 Web3 应用中，多链切换和钱包断连处理主要通过监听 MetaMask（或其他钱包）提供的事件来实现：

1. **多链切换**：监听 `chainChanged` 事件
2. **钱包断连**：监听 `accountsChanged` 事件

---

### 二、多链切换实现

#### 2.1 代码实现

```javascript
// 监听网络切换
useEffect(() => {
  if (!window.ethereum) return;

  const handleChainChanged = (newChainId) => {
    console.log('网络切换:', newChainId);
    setChainId(newChainId);
    
    // 网络切换后重新获取余额
    if (account) {
      fetchBalance(account);
    }
  };

  // 监听网络变化事件
  window.ethereum.on('chainChanged', handleChainChanged);

  // 清理函数：移除监听器
  return () => {
    window.ethereum.removeListener('chainChanged', handleChainChanged);
  };
}, [account, fetchBalance]);
```

#### 2.2 实现要点

1. **事件监听**：使用 `window.ethereum.on('chainChanged', handler)`
2. **状态更新**：切换后更新 `chainId` 状态
3. **数据刷新**：重新获取账户余额（不同链的余额不同）
4. **清理函数**：组件卸载时移除监听器，防止内存泄漏
5. **依赖管理**：将 `account` 和 `fetchBalance` 加入依赖数组

#### 2.3 网络识别

```javascript
const getNetworkName = (chainId) => {
```

### 三、钱包断连处理

#### 3.1 代码实现

```javascript
// 监听账户切换
useEffect(() => {
  if (!window.ethereum) return;

  const handleAccountsChanged = (accounts) => {
    console.log('账户切换:', accounts);
    
    if (accounts.length === 0) {
      // 用户断开了所有账户 - 触发断连逻辑
      disconnect();
    } else if (accounts[0] !== account) {
      // 切换到新账户
      setAccount(accounts[0]);
      fetchBalance(accounts[0]);
    }
  };

  window.ethereum.on('accountsChanged', handleAccountsChanged);

  return () => {
    window.ethereum.removeListener('accountsChanged', handleAccountsChanged);
  };
}, [account, disconnect, fetchBalance]);

// 断开连接函数
const disconnect = useCallback(() => {
  setAccount(null);
  setBalance('0');
  setChainId(null);
  setError(null);
}, []);
```

#### 3.2 实现要点

1. **事件监听**：使用 `window.ethereum.on('accountsChanged', handler)`
2. **断连判断**：`accounts.length === 0` 表示用户断开了钱包
3. **状态清理**：清空所有相关状态（account、balance、chainId、error）
4. **账户切换**：同时处理用户切换账户的情况
5. **性能优化**：使用 `useCallback` 缓存 disconnect 函数

### 四、完整的 useWallet Hook

```javascript
import { useState, useEffect, useCallback } from 'react';

const useWallet = () => {
  // 状态管理
  const [account, setAccount] = useState(null);
  const [balance, setBalance] = useState('0');
  const [chainId, setChainId] = useState(null);
  const [isConnecting, setIsConnecting] = useState(false);
  const [error, setError] = useState(null);

  // 获取账户余额
  const fetchBalance = useCallback(async (address) => {
    try {
      if (!window.ethereum) return;
      
      const balance = await window.ethereum.request({
        method: 'eth_getBalance',
        params: [address, 'latest']
      });
      
      // 将 Wei 转换为 ETH
      const ethBalance = parseInt(balance, 16) / Math.pow(10, 18);
      setBalance(ethBalance.toFixed(4));
    } catch (err) {
      console.error('获取余额失败:', err);
      setError(err.message);
    }
  }, []);

  // 连接钱包
  const connect = useCallback(async () => {
    if (!window.ethereum) {
      setError('请安装 MetaMask 钱包');
      return;
    }

    setIsConnecting(true);
    setError(null);

    try {
      const accounts = await window.ethereum.request({
        method: 'eth_requestAccounts'
      });

      const chainId = await window.ethereum.request({
        method: 'eth_chainId'
      });

      setAccount(accounts[0]);
      setChainId(chainId);
      await fetchBalance(accounts[0]);
    } catch (err) {
      console.error('连接钱包失败:', err);
      setError(err.message);
    } finally {
      setIsConnecting(false);
    }
  }, [fetchBalance]);

  // 断开连接
  const disconnect = useCallback(() => {
    setAccount(null);
    setBalance('0');
    setChainId(null);
    setError(null);
  }, []);

  // 监听账户切换（包含断连处理）
  useEffect(() => {
    if (!window.ethereum) return;

    const handleAccountsChanged = (accounts) => {
      if (accounts.length === 0) {
        disconnect();
      } else if (accounts[0] !== account) {
        setAccount(accounts[0]);
        fetchBalance(accounts[0]);
      }
    };

    window.ethereum.on('accountsChanged', handleAccountsChanged);

    return () => {
      window.ethereum.removeListener('accountsChanged', handleAccountsChanged);
    };
  }, [account, disconnect, fetchBalance]);

  // 监听网络切换
  useEffect(() => {
    if (!window.ethereum) return;

    const handleChainChanged = (newChainId) => {
      setChainId(newChainId);
      if (account) {
        fetchBalance(account);
      }
    };

    window.ethereum.on('chainChanged', handleChainChanged);

    return () => {
      window.ethereum.removeListener('chainChanged', handleChainChanged);
    };
  }, [account, fetchBalance]);

  return {
    account,
    balance,
    chainId,
    isConnecting,
    error,
    isConnected: !!account,
    connect,
    disconnect,
    refreshBalance: () => account && fetchBalance(account)
  };
};

export default useWallet;
```

### 五、使用示例

```javascript
function WalletComponent() {
  const {
    account,
    balance,
    chainId,
    isConnected,
    connect,
    disconnect
  } = useWallet();

  return (
    <div>
      {!isConnected ? (
        <button onClick={connect}>连接钱包</button>
      ) : (
        <div>
          <p>账户: {account}</p>
          <p>余额: {balance} ETH</p>
          <p>网络: {chainId}</p>
          <button onClick={disconnect}>断开连接</button>
        </div>
      )}
    </div>
  );
}
```

### 六、注意事项和最佳实践

#### 6.1 事件监听清理

```javascript
// ✅ 正确：使用清理函数
useEffect(() => {
  const handler = (accounts) => { /* ... */ };
  window.ethereum.on('accountsChanged', handler);
  
  return () => {
    window.ethereum.removeListener('accountsChanged', handler);
  };
}, [dependencies]);

// ❌ 错误：没有清理，会导致内存泄漏
useEffect(() => {
  window.ethereum.on('accountsChanged', handler);
}, [dependencies]);
```

```javascript
// 同时查询多条链的余额
export function useMultiChainBalance(chains = [1, 56, 137]) {
  const { address } = useWallet();
  
  // 为每条链创建查询
  const balanceQueries = chains.map(chainId => 
    useGetBalanceQuery(
      { address, chainId },
      { skip: !address }
    )
  );
  
  // 聚合所有链的余额
  const totalBalance = balanceQueries.reduce((sum, query) => {
    return sum + (query.data?.amountInEth || 0);
  }, 0);

  return {
    balances: balanceQueries.map((q, i) => ({
      chainId: chains[i],
      balance: q.data?.amountInEth || 0,
      isLoading: q.isLoading,
    })),
    totalBalance,
    isLoading,
  };
}
```

### 八、常见面试追问

#### Q1: 为什么要在清理函数中移除事件监听器？

**A:** 防止内存泄漏。如果不移除监听器：
- 组件卸载后，监听器仍然存在
- 事件触发时会尝试更新已卸载组件的状态
- 导致内存泄漏和 "Can't perform a React state update on an unmounted component" 警告

#### Q2: 为什么 chainChanged 后要重新获取余额？

**A:** 因为：
- 不同链的账户余额是独立的
- 同一个地址在不同链上的余额不同
- 切换链后需要获取新链上的余额数据

#### Q3: 如果用户快速切换多次链，如何避免竞态条件？

**A:** 可以使用以下方法：
```javascript
const fetchBalance = useCallback(async (address, chainId) => {
  const requestId = Date.now();
  lastRequestIdRef.current = requestId;
  
  const balance = await getBalance(address, chainId);
  
  // 只有最新的请求才更新状态
  if (requestId === lastRequestIdRef.current) {
    setBalance(balance);
  }
}, []);
```

#### Q4: 如何处理用户拒绝连接钱包的情况？

**A:** 在 connect 函数中捕获错误：
```javascript
try {
  const accounts = await window.ethereum.request({
    method: 'eth_requestAccounts'
  });
} catch (err) {
  if (err.code === 4001) {
    // 用户拒绝连接
    setError('用户拒绝连接钱包');
  } else {
    setError(err.message);
  }
}
```

## 🎯 面试回答要点总结

1. **多链切换**：监听 `chainChanged` 事件，更新 chainId，重新获取余额
2. **钱包断连**：监听 `accountsChanged` 事件，判断 accounts 数组长度为 0
3. **清理函数**：必须移除事件监听器，防止内存泄漏
4. **依赖数组**：正确管理 useEffect 的依赖项
5. **错误处理**：捕获异常，提供友好的错误提示
6. **性能优化**：使用 useCallback 缓存函数，避免不必要的重渲染
